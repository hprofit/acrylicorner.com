Basic Game Lifecycle



let gameIsPlaying = true;
startup();
while (gameIsPlaying) {
    processInput();
    updateEntities();

Start by Rendering

While it's the last step in the game loop process, it's the best starting point because from there you can see your entity updates and responses to input represented visually on the screen.

Z-Index

Despite being a 2D game, entities will still have a z coordinate known as a Z-Index. The closer the Z-Index to the camera, the less likely it is to be drawn over by another entity.
This can also effectively split your 2D games into layers:

  • Background
  • Foreground
  • Entities
  • HUD

These layers can be rearranged or have layers added for visual effects as needed.

Processing Input



class Player {
    ...
    handleInput(keys) {
        // Reset the players velocity (movement)
        this.velocity.x = 0;

        if (keys.left.isDown) {
            // Move to the left
            this.velocity.x = -150;
        }
        else if (keys.right.isDown) {
            // Move to the right
            this.velocity.x = 150;
        }
        if (keys.up.isDown) {
            this.velocity.y = -350
        }
    }
}
