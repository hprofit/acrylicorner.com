Deep Learning

One of the most complex types of AI, deep learning aims to solve the problem of teaching a machine to do something rather than telling it how.
The implications this has are far reaching but it is a hugely complex task.

Neural Networks

One of the common methods of performing machine learning is through the use of neural networks.

Neural networks are made to emulate the human brain.

How does it work?

A neural net is comprised of layers and neurons.

Given some input and an expected output, the network is trained neuron by neuron to give the correct output even in new scenarios.

Conway's Game of Life

A form of cellular automaton, it's less of a game and more of a living experiment!
Generally the game is played based off an initial input and from there the player sits back and watches how the cells act over time.

How does it work?

Simple Rules to follow

  • Living cells with < 2 living neighbors die as through from loneliness
  • Living cells with 2 to 3 living neighbors remain alive
  • Living cells with > 3 living neighbors die as through from over crowding
  • Dead cells with 2 living neighbors spawns as through from reproduction

Flocking

Simulating nature through a set of mathematical rules, flocking AI demonstrates how a mass of entities would move about in the wild.

How does it work?

Each entity in a flock has a set of goals it tries to reach

  • Stay close to neighbors
  • Stay out of their neighbors' personal space
  • Head in the same direction as their neighbors
  • Avoid walls and obstacles
  • Avoid predators or enemies

Pathfinding via A*

Pathfinding using A* is a quick way to find the shortest route from point A to B along a series of interconnected nodes.

The process involves the use of a min-heap object and a heuristic to help steer the AI away from paths that are too long.

How does it work?

A min-heap is a binary tree where the parent's value is less than both children.
Utilizing a min-heap, A* begins with a start and goal point in mind and a grid of connected nodes.

For the current node, it calculates distance between itself and the goal. If the result is within acceptable parameters, it is added as a viable node to consider later.

This process repeats until the algorithm finds the end node. From there, it reconstructs the path based off of the viable nodes stored from earlier.
